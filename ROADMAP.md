## Context for the executor

- **Stack:** Next.js 15 App Router + React 19 + Tailwind CSS 4 + TypeScript. No test suite; the verification gate is `npm run typecheck && npm run build` (both currently pass — keep them passing). Deps are installed (`npm ci` if node_modules is missing).
- **Data model:** everything is intentionally **mock/local** — courses live in `src/lib/data.ts` (all content trilingual `{fr,en,ar}`), auth is hardcoded users in `src/app/api/auth/*/route.ts`, enrollment/progress in localStorage (`src/lib/progress.ts`). **Do NOT add a database, real auth, or external services** — the redesign brief (DESIGN_PROMPT.md) explicitly says the design should not assume backend constraints; keep mock data.
- **Redesign brief:** `DESIGN_PROMPT.md`. Key points: distinctive teen-friendly identity, design tokens as CSS variables, light+dark mode, missing screens (student dashboard, catalog with filters, quizzes, profile/settings, instructor pages), full **RTL mirroring for Arabic**, WCAG AA, mobile-first.
- **Trilingual rule (never break):** every user-visible string goes through `src/lib/i18n.ts` and must be added in **all three** locales (fr/en/ar) — the `TranslationKeys` type enforces this. Never hardcode UI text in components.
- **RTL rule:** prefer logical Tailwind utilities (`ms-`/`me-`/`ps-`/`pe-`/`start-`/`end-`) over `ml-`/`mr-`/etc. so Arabic mirrors correctly.
- Layout/serving of this app is not wired to any homelab service — building is safe; do not touch anything outside this repo.
